## Architecture decisions for the .NET build (from CLAUDE.md + dotnet skill)
- Clean Architecture (API→Application→Domain; Infra→App+Domain; Domain refs nothing).
- Custom CQRS — NO MediatR. Hand-rolled ICommand/IQuery dispatchers from DI.
- Database-first EF Core 10 — scaffold/map entities from canonical schema; migrations only track drift.
- Custom CQRS dispatcher, pipeline behaviors (validation, logging, audit, idempotency).
- YARP gateway (JWT validation + rate limiting at edge), Aspire AppHost, RabbitMQ integration events, OpenTelemetry, Serilog.
- DRY: shared DTOs/enums → mediq.SharedDataModel; cross-cutting → mediq.Utilities. SEARCH before adding.
- RBAC: resolve-once-per-request, in-memory checks. Backend-driven menus. No hardcoded roles. security-compliance-auditor has VETO per slice.
