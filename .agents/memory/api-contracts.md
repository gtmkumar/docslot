## Slice 02 — platform_api (Platform-as-a-Service, developer portal). DTOs in `mediq.SharedDataModel/Docslot/PlatformApi/`.
OAuth 2.0 client-credentials + scoped JWTs + webhooks. Two auth schemes coexist: USER tokens carry `permissions` (RBAC), CLIENT tokens carry `scope` (a `token_use=client` JWT claim, space-delimited per OAuth). Endpoints:
- `POST /api/v1/oauth/token` (anon) → `OAuthTokenResponse {accessToken, tokenType:"Bearer", expiresIn, scope}`. Body `OAuthTokenRequest {grantType:"client_credentials", clientId(=client_code), clientSecret, scope?(space-delim), tenantId?}`. Bad secret/unapproved → 403 `invalid_client`; ungranted scope → 422 `invalid_scope`.
- `POST /api/v1/oauth/revoke` (anon) → idempotent revoke by token hash.
- `GET /api/v1/public/bookings|patients` — third-party surface gated by `[RequireScope("docslot.bookings.read"|"docslot.patients.read")]` (in-memory, resolve-once). Lacking scope → 403.
- Admin (gated `platform.api_clients.manage`): `GET/POST /api/v1/api-clients`, `GET /api/v1/api-scopes`, `POST /api/v1/api-clients/{id}/rotate-secret|status`, `PUT .../scopes|rate-limits`. Register/rotate return `ApiClientSecretResult {clientId, clientCode, clientSecret}` — plaintext ONCE.
- Webhooks (gated `platform.api_clients.manage`): `GET /api/v1/webhooks/event-types`, `GET/POST /api/v1/webhooks`, `PUT/DELETE /api/v1/webhooks/{id}`, `POST /api/v1/webhooks/publish` (synthetic event trigger, slice 02). Create returns `CreateWebhookResult {webhookId, signingSecret}` — plaintext ONCE. Delivery sets header `X-DocSlot-Signature: sha256=<hex>` = HMAC-SHA256(rawBody, secret); subscriber recomputes to verify.
DTOs: `OAuthTokenRequest/Response`, `OAuthRevokeRequest`, `ScopeDto`, `ApiClientDto`, `RegisterApiClientRequest`, `ApiClientSecretResult`, `Set{ClientScopes,ClientStatus,ClientRateLimits}Request`, `WebhookSubscriptionDto`, `CreateWebhookRequest/Result`, `UpdateWebhookRequest`, `WebhookDeliveryDto`, `EventTypeDto`. Secrets/tokens NEVER returned after creation; client secret + token stored as bcrypt/SHA-256 hashes, webhook secret AES-encrypted (recoverable to sign).
