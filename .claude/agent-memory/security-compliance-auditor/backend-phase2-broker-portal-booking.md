Phase-2 organic attribution path #2: BROKER-PORTAL BOOKING. A logged-in broker books on behalf of a referred patient via POST /api/v1/commission/me/bookings → BEHALF booking (consent 'pending', DPDP OTP) + AUTO-VERIFIED `broker_portal_booking` attribution, in ONE tenant-scoped UoW. Audited 2026-06-28 (working tree, branch feat/phase2-broker-portal-booking, uncommitted) — **VERDICT: PASS, no conditions.** Build clean (only pre-existing MessagePack NU1903); full suite 177/177; 38/38 targeted (BrokerPortal+CommissionPipeline+PostHoc+Consent); live DB verified.

- DOUBLE-DEBIT IMPOSSIBLE: all 3 filter on NON-terminal status and flip to terminal 'reversed' (excluded from all 3 filters) → first to lock wins, others match 0 rows. Booking cancel itself status-guarded (won't double-free slot). Sweep-vs-late-in-band-reply race resolves to one debit (mirror of post-hoc claim sweep-vs-deny, already verified). EXACTLY-ONCE holds.

**Gate 2 — broker can NEVER earn on a booking the patient never consented to (THE money-safety crux):**
- `broker_portal_booking` ⇒ verification 'auto_verified' (AttributionEngine.cs:70-72) ⇒ earns on booking COMPLETION via MarkEarnedForBookingAsync (filter commission_status='pending' AND verification IN auto_verified/patient_confirmed/admin_override). NO patient confirmation of the *attribution* needed (broker demonstrably booked).
- BUT earning requires the BOOKING to reach 'completed', and the state machine (Booking.cs:132-142) has NO Pending→Completed edge. Only path: Pending→Confirmed (Approve) →[CheckedIn]→ Completed. **Approve is BLOCKED by the DPDP consent guard**: BookingActionCommand.cs:51 throws if `AwaitingPatientConsent` (= behalf && consent != confirmed). Consent only becomes 'confirmed' via the PATIENT's own OTP reply (TryVerifyReplyAsync → ConfirmPatientConsent, which itself requires behalf+pending). OnBookingCompletedAsync/MarkEarned reachable ONLY from BookingActionCommand.Complete (grep-verified sole caller). ⇒ unconsented booking can't complete ⇒ can't earn. On deny/expiry all 3 cancel paths reverse → pending cleared. **No earn-without-consent path exists.**
