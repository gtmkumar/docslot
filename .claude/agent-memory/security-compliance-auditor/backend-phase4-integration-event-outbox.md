**NEW: `platform_api.integration_event_outbox` (TABLE 27).** Columns: outbox_id PK, event_id UUID NOT NULL UNIQUE (the inline UNIQUE auto-creates the index that `ON CONFLICT (event_id) DO NOTHING` targets — the explicit dedup index was removed, dedup still enforced), event_type (NOT FK'd to api_event_types — intentional, capture survives registry drift), tenant_id NULLABLE FK→tenants (locality/forensics only, NOT a security boundary), payload JSONB (IDs/tokens ONLY), status CHECK in (pending/processing/success/failed/abandoned), attempt_count, next_retry_at (backoff schedule AND processing-lease watermark), last_error, published_at. Indexes: idx_integration_outbox_due (partial WHERE status in pending/failed/processing), idx_integration_outbox_tenant.

**Event-PHI = CLEAN (verified at the source, all 13 call sites).** Payload is an arbitrary `object payload` whose safety depends entirely on each caller. I traced EVERY caller of IBookingEventPublisher + IBrokerEventPublisher: booking create/action/reschedule, prescription issued/amended, report uploaded/delivered, ABDM record-linked, commission earned/reversed, attribution created, payout paid. ALL carry IDs (patient_id/booking_id/report_id/record_id/prescription_id/broker_id/attribution_id/slot_id) + amounts (commission_inr/net_inr) + status/reason/reference CODES only. ABDM payload explicitly EXCLUDES care_context_id/ABHA. The only free-text-capable field (commission.reversed `reason`) is sourced ONLY from hardcoded literals (no_response/patient_denied/failed_attempts/superseded_by_consent) — never caller free text. No patient_name/phone/diagnosis/PAN/bank in any payload.
