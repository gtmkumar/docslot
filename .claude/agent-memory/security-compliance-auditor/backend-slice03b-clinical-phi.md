**LIVE-DB VERIFIED (not just code):**
- LEAST-PRIVILEGE ROLE REAL: `docslot_app` exists, `rolsuper=f rolbypassrls=f rolcreatedb=f rolcreaterole=f rolcanlogin=t`. App connects as it (mediq.Api + AppHost appsettings `Username=docslot_app`). Live-tested: `docslot_app` gets `permission denied` on UPDATE/DELETE of `audit_log` AND `audit_chain` (grant-level), and `SET app.allow_audit_maintenance='on'` does NOT help it (still denied) — `block_audit_log_mutation` checks `current_user <> 'docslot_app'` (`01_platform_core.sql:580`). Belt-and-suspenders (grant + trigger).
- RLS ENFORCED: all 5 PHI tables `relrowsecurity=t` (relforcerowsecurity=f, but tables owned by `gtmkumar` and `docslot_app` is NOT a member → non-owner → RLS APPLIES to the app role). Policy `tenant_isolation_*` FOR ALL `USING (tenant_id = current_tenant_id() OR current_is_super_admin())`; WITH CHECK null → USING used for INSERT too (can't insert cross-tenant). docslot_app has SELECT/INSERT/UPDATE only (NO DELETE → soft-delete). Live: as docslot_app with app.tenant_id set, prescriptions visible only for that tenant. Genuine test `RLS_Blocks_Cross_Tenant_Clinical_Read` inserts tenant-B row as admin, asserts docslot_app scoped to A sees 0, admin sees 1.
- READ PATH SETS THE GUC: `TenantScopeQueryBehavior` (`Cqrs/Behaviors.cs:39`) registered for ALL queries (`DependencyInjection.cs:37`), calls `SetTenantScopeAsync` before every read so RLS applies on reads too (not just command UoW). NOTE the read path uses `set_config(..., is_local=false)` = SESSION scope (not SET LOCAL) and pins the EF connection open for the scoped DbContext (queries run outside a tx) — `UnitOfWork.cs:16-29`. Pool-safety relies on Npgsql's default DISCARD ALL on connection return (the in-code comment claims a defensive reset but I found NO explicit reset code — DISCARD ALL is the actual mechanism). VERIFY this stays correct if Npgsql `No Reset On Close` is ever set or multiplexing enabled — that would be a cross-tenant GUC-leak regression. Tracked watch-item, not a blocker (tests pass, default reset active).
- ENCRYPTION AT REST: prescriptions diagnosis/medications/examination/chief_complaints, lab_reports.structured_results, medical_history.title/description, abdm.fhir_bundle all encrypted on write (FieldEncryptionService AES-256-GCM from Slice 05), decrypted only after consent+purpose pass. Test reads raw column, asserts plaintext absent. key_usage_log columns carry NO plaintext PHI.
- CONSENT + PURPOSE ON EVERY CLINICAL ENDPOINT: ClinicalController `Purpose()` → 422 if `X-Purpose-Of-Use` absent; handlers re-check + log to purpose_of_use_log before returning. Consent: prescriptions/reports/history check `patient.HasActiveConsent`→403; ABDM additionally requires `AbdmConsentService` granted/unexpired/non-revoked for (patient, requesting_tenant)→403. Each endpoint gated by a DISTINCT real seed key (prescription.create/read, report.upload/read, medical_history.read, abdm.records.create/read).
- AUDIT-CHAIN RACE FIXED: `pg_advisory_xact_lock(8675309001)` in `append_to_audit_chain` (`05_security_hardening.sql:189`) serializes appenders. 12-parallel-write test asserts 0 breaks. Live chain 895 rows, 0 breaks (grew from Slice 05's 547 through concurrent tests — lock holds).
