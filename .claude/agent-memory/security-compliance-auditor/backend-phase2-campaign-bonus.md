- **refund_campaign_bonus_on_reversal** (live prosecdef=TRUE=DEFINER, proconfig search_path=commission,pg_temp, owner gtmkumar): AFTER UPDATE OF commission_status WHEN NEW IN(reversed,rejected) AND OLD.commission_status IS DISTINCT FROM NEW (note: prompt said `OLD IS DISTINCT FROM NEW`, actual code is the stronger column-level guard). Refunds source_metadata->>campaign_bonus_inr to budget, GREATEST(0,...) underflow guard. SECURITY DEFINER safe: **trigger fn CANNOT be called directly** (`ERROR: trigger functions can only be called as triggers` — verified live), so PUBLIC execute is non-exploitable; pinned search_path (pg_temp LAST) blocks injection; tenant correctness inherent (campaign_id matched to attribution's tenant at grant time, source_metadata written server-side ONCE at insert, no client field flows in, never UPDATEd post-insert).
- **Every reversal path refunds exactly once**: MarkReversedForBookingAsync (cancel/no-show via OnBookingReversedAsync), ReverseOneAsync (dispute clawback RulesDisputesCampaigns + patient-deny CommissionLifecycleService), expire_stale_attribution_claims SQL sweep — ALL set commission_status='reversed' via UPDATE → fires trigger once. All reversal WHERE/CTE filter `IN('pending','earned','ready_to_pay'[,'paid'])` so already-reversed never re-selected → no double-refund. reversed→reversed: OLD=NEW → IS DISTINCT FROM false → no fire.
- **source_metadata '{}' EF fix**: literal '{}' in ExecuteSqlRaw misread as {0} placeholder → C# defaults to "{}" string param @p11::jsonb. Correct; verified parameterised (no injection).
- **Lifecycle folds correctly**: bonus folded into commission_amount_inr → rides pending→earned→ready_to_pay→paid + same wallet buckets (ApplyAttributed credits TOTAL). ₹100 GROSS payout floor (v_ready_payouts HAVING SUM(commission_amount_inr)>=100 AND PayoutCalculator.ReadyToPayGross) sums total incl bonus on BOTH sides — consistent, broker genuinely owed total.
- **PHI-free events**: commission.attribution.created adds bonus_inr + campaign_id (UUIDs/amounts, no patient PHI). Audit ChangeSummary records base+bonus split.
